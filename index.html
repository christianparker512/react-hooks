What is the difference between a class component and a functional component?

(Common interview question)

  • class components use classes (this) and functional components are just functions
  • class components have lifecycle methods (render, componentDidMount, componentDidUpdate, etc)
  • functional components are stateless

What are React Hooks?

Syntactic Benefits:

  • no more deciding whether to write a function or a class component / convert later
  • allows us to ditch all uses of class components, this, and method bindings (this.handleClick = this.handleClick.bind(this))

What are React Hooks?

Logical Benefits:

  • blocks of logic are easier to reason about since they're just functions instead of being spread / intertwined across lifecycle methods.

Render Props

  • A component with a render prop takes a function that returns a React element and calls it instead of implementing its own render logic
  • Hard to distinguish between regular components and render-prop components that are used for sharing logic
  • sooner or later you’ll face a problem of nesting components within multiple render prop callbacks: render props callback hell

Common Hooks - useCallback

  • Returns a memoized callback (callback that only changes if one of the dependencies has changed)
  • Despite name, only use when referential equality is checked

Common Hooks - useCallback

  • Used as a callback ref - run some code when React attaches or detaches a ref to a DOM node
  • useRef doesn’t notify us about changes to the current ref value
  • useful for measuring DOM nodes

Common Hooks - useMemo

  • used to memoize computationally expensive calculations

Common Hooks - useRef

  • Used to hold a mutable value (or function) that persists render to render and, when changed, does NOT trigger a re-render.
  • Can be thought of as class instance variables / methods
  • Useful for working with web apis (setInterval, addEventListener)

Best Practices - Rules of Hooks

  • Only call hooks at the Top Level - no loops, conditions, callback functions [place all hooks together at the top of a component]
  • Only call hooks from react functions - React function components and custom Hooks
  • Use the eslint-plugin-react-hooks to enforce these two rules
